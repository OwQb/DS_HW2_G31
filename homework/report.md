## 效能分析
### Max/Min Heap
1. **IsEmpty()**   
  這個函式只做一個簡單的比較  
  沒有用到迴圈或遞迴，僅花費固定時間  
  因此時間複雜度是 O(1)

2. **Top()**  
  這個函式只是回傳 heap[1]（堆頂元素）  
  不需要遍歷堆，也沒有移動元素  
  因此時間複雜度是 O(1)  

3. **Push(const T&)**  
  插入時，先把新元素放到最後一個位置（O(1)）  
  然後進行「上浮調整」：  
  每次與父節點比較並可能交換（最多log(n)次）  
  因為堆是完全二元樹，樹高最多log(n)  
  因此時間複雜度為 O(log n)  

4. **Pop()**  
  移除堆頂後，最後一個元素 lastE 被提到頂部  
  然後進行「下沉調整」：  
  每次與較小的子節點比較並可能交換  
  最多調整 log(n) 層  
  因此時間複雜度為 O(log n)  

### Binary Search Tree
1. **為何繪製比率高度 log2n 作為 n 的函數。此比率應大致恆定（約2）?**  
   若將 n 個節點隨機插入二元搜尋樹，所形成的 BST 是隨機形狀的 BST，  
   理論上，隨機插入下的 BST 平均高度為 O(log n)，實際上其常數因子約為 c ≈ 4.311（但中位數高度的常數約為 c ≈ 2）。  
   log(n) 是最佳情況下平衡 BST 的高度（例如 AVL 或紅黑樹的高度下界），  
   隨機插入會導致 BST 高度略高於 log(n)，但平均仍維持在相同階數，  
   因此高度除以 log(n)，也就是：比率 = 高度 / log(n)  
   將會是個約略恆定的數值，通常介於 1.3 到 2.5，根據不同的 n 值會略有波動。
   所以繪製比率高度 log2n 作為 n 的函數。此比率應大致恆定（約2）。  
    
2. **從二元搜尋樹中刪除鍵為k的對時間複雜度應是多少?**  
   刪除時會先搜尋目標節點 key ，其時間複雜度是與高度成正比，隨後可能需要找後繼節點（即右子樹最小值） 也是一條向下走的路徑，最後更新指標、刪除節點。  
   若樹的高度為 h，則：搜尋、插入、刪除的最壞時間複雜度皆為 O(h)，在隨機 BST 中，h ≈ c * log(n)，  
   因此刪除的期望時間為 log(n)。
### 緩衝區演算法計算
1. 外部排序第二回合的目標是把所有 m 個行程透過多次 k-way merge 合併成一個完整排序的行程。假設一次合併 k 個行程，就會有：  
合併次數（rounds）≈ ⌈log<sub>k</sub> m⌉<br><br>
總輸入時間 t_input，也就是把所有 n 筆資料從磁碟讀入主記憶體所花費的時間，    
每次從磁碟讀一個區塊的時間包含：  
搜尋時間 t<sub>s</sub>  
延遲時間 t<sub>1</sub>  
傳送時間 n ． t<sub>t</sub>  
總共需要讀取 n 筆資料，在合併過程中每筆資料都要至少讀一次。<br><br> 
所以總輸入時間為：t<sub>input</sub> = ( ${n} \over  {B}$ ) ． (t<sub>s</sub> + t<sub>1</sub>) + n ． t<sub>t</sub>
其中：  
B 是區塊大小（每次讀入多少筆記錄）  
${n} \over  {B}$ 是總共要進行多少次磁碟 I/O 區塊讀取  
但根據題意，記憶體大小為 S，且分為 k 個輸入緩衝區 + 1 個輸出緩衝區，所以每個緩衝區大小約為：  
B = ${S} \over  {k+1}$  
代入後，輸入區塊數量為：  
${n} \over  {B}$ = ${n(k+1)} \over  {S}$<br><br> 
**因此總輸入時間為：  
t<sub>input</sub>(k) = ${n(k+1)} \over {S}$ ． (t<sub>s</sub> + t<sub>1</sub>) + n ． t<sub>t</sub><br><br>**
2. 根據題目已知  
t<sub>s</sub>  = 80 ms = 0.08 秒  
t<sub>1</sub>  = 20 ms = 0.02 秒  
t<sub>t</sub>  = $10^{-3}$ = 0.001秒/記錄  
n = 200000  
S = 2000（記憶體可放 2000 筆記錄）<br><br>
代入總輸入時間公式：  
t<sub>input</sub>(k) = ${200000(k+1)} \over {2000}$ ． (0.02 + 0.08) + 200000 ． 0.001  
化簡：  
t<sub>input</sub>(k) = 100(k+1)(0.1)+200 = 10(k+1)+200 = 10k+210 秒<br><br>
檢驗是否存在使得 t<sub>cpu</sub> ≈ t<sub>input</sub> 的 k 值可以先假設：
t<sub>cpu</sub> = 500 秒  
帶入公式：  
t<sub>input</sub>(k) ⇒ t<sub>cpu</sub> 10k+210=500 ⇒ k = ${290} \over {10}$ = 29<br><br>
**得知當t<sub>cpu</sub>=500時 k = 29 ，所以的確存在某個 k 值使得 t_input ≈ t_cpu，這個 k 值可以透過解方程來找出，並達成 CPU 與 I/O 負載平衡**

## 申論及開發報告

### 申論
在本次作業中，我們針對三個主要題目進行了理論分析與程式實作，並透過實際測試驗證了演算法與資料結構的時間複雜度與效能表現。以下針對每個題目進行綜合性的探討與結論。

#### 題目一：Max/Min Heap 實作
Max Heap（最大堆）與 Min Heap（最小堆）皆屬於完全二元樹的陣列實作，主要操作有 IsEmpty()、Top()、Push(const T&) 與 Pop()。透過理論推導可知：
IsEmpty() 與 Top() 僅涉及單一記憶體存取或變數比較，因此時間複雜度都是 O(1)。
Push(const T&) 涉及將新元素插入尾端，並執行「上浮」（percolate-up）操作，最多需要比較並交換至樹高高度──約 log n 層，因此時間複雜度為 O(log n)。
Pop()（刪除堆頂）則是將最後一個元素搬到頂部，並執行「下沉」（percolate-down）操作，同樣最多比較並交換 log n 次，故時間複雜度為 O(log n)。
我們在 n = 1,000、2,000、3,000、…、10,000 等不同規模下，以隨機值逐一填滿 Heap，並重複執行 Push/Pop 等操作十萬次，取得平均運算時間。實驗結果顯示，隨著 n 增加，平均 Push/Pop 的執行時間與 log₂n 的趨勢相符，中間少數異常值可能來自作業系統排程或快取行為，但整體趨勢穩定且符合預期。這驗證了設計上 Heap 操作在不同大小下確實維持 O(log n) 的成長特性，並且 IsEmpty()、Top() 始終維持常數時間。

#### 題目二：Binary Search Tree 實作
當 n 個節點以均勻隨機方式依序插入空的二元搜尋樹（BST）時，理論上此隨機 BST 的平均高度為 O(log n)，但常數因子大於完美平衡樹約略 1：實際平均高度約介於 (1.3 × log n) 至 (2.5 × log n) 之間，視 n 的大小而略有波動。本作業中，我們令 n = 100、500、1,000、2,000、3,000、…、10,000，對每個 n 值隨機產生 n 個不重複鍵值插入 BST，重複多次取平均，並計算實際高度 h 與 log n 之比率 r = h / log n。實驗結果顯示，r 隨 n 變化時，大致維持在 1.8 ～ 2.2 的區間，整體略微振盪但趨於穩定，符合「隨機 BST 平均高度與 log n 成正比且常數因子約 2」的理論預期。
至於「BST 刪除鍵為 k 的節點」之時間複雜度分析：刪除步驟首先需搜尋目標節點，時間與樹高 h 成正比；若該節點有兩個子節點，則需找其右子樹最小節點（後繼），再以該節點的鍵值替換，最後刪除該後繼節點，整體仍為樹高範圍內的遞迴與指標更新。因此，刪除操作的最壞時間複雜度為 O(h)。在本次隨機 BST 的上下文中，h ≈ c · log n（c 約 1.3 ～ 2.5），故刪除的期望時間為 O(log n)。實驗中透過每次隨機刪除測量時間，結果與 log n 成正比，進一步印證理論分析。
#### 題目三：緩衝區演算法分析
在外部排序第二回合中，若要將 m 個排序行程（runs）經過 k-way merge 合併成一個排序行程，且主記憶體可用於輸入/輸出的緩衝區大小為 S 筆記錄，則每個緩衝區可分配 B = ${S} \over  {k+1}$ 筆記錄作為緩衝。總共要讀取 n 筆記錄，則磁碟 I/O 次數約為 ${n} \over  {B}$ = ${n(k+1)} \over  {S}$次。每次 I/O 需花費搜尋時間 t<sub>s</sub>、延遲時間 t<sub>1</sub>，故總輸入時間可表示為：
t<sub>input</sub>(k) = ${n(k+1)} \over {S}$ ． (t<sub>s</sub> + t<sub>1</sub>) + n ． t<sub>t</sub><br><br>
其中 tₜ 為單筆記錄傳輸時間。在本實驗中，代入參數、t<sub>s</sub>  = 80 ms = 0.08 秒、t<sub>1</sub>  = 20 ms = 0.02 秒、t<sub>t</sub>  = $10^{-3}$ = 0.001秒/記錄、n = 200000、S = 2000（記憶體可放 2000 筆記錄），可得：  
t<sub>input</sub>(k) = ${200000(k+1)} \over {2000}$ ． (0.02 + 0.08) + 200000 ． 0.001  
= 100(k+1) · 0.1 + 200  
= 10(k+1) + 200  
= 10k + 210 (秒)  
若 CPU 合併時間 t<sub>cpu</sub> 為常數（與 k 無關），當 t<sub>cpu</sub> = 500 秒時，方程式 10k + 210 = 500 解得 k ≈ 29。實驗結果繪出 t<sub>input</sub>(k) 隨 k 變化的曲線後，確實可看到在 k ≈ 29 時，t<sub>input</sub> 與 t<sub>cpu</sub> 相當，意味著 I/O 與 CPU 負載得到平衡。若 k 過小，I/O 次數增加，t<sub>input</sub> 過高；k 過大，雖可減少 I/O 次數，卻縮小每個緩衝區 B，導致每次 I/O 讀入的記錄數減少，i/o 負擔並不成比例地降低；因此存在一個使 I/O 與 CPU 負載平衡的最佳 k 值。此結論與理論推導吻合。
